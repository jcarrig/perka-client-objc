Detailed Description

A Clerk represents either an employee identity or a role identity used by the validator app.

Two distinct classes of Clerks exist:

  • Named clerks, representing a user authorized to perform actions at zero or more locations
  • Anonymous clerks, representing an access role for a single location

A named clerk must downgrade to an anonymous clerk in order to perform validations at a single location. This avoids the need to store any particular user's credentials in the validator app.
